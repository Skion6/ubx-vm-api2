Capabilities

Built for Scale

Every VM runs in a hardened Docker container with strict resource limits and automatic lifecycle management.

⚡

Instant Provisioning

Full Ubuntu desktops spin up in seconds via a single GET request. No warm-up, no waiting.

🔒

Isolated Containers

Each VM is a sandboxed Docker container with its own filesystem, network, and process space.

🌐

Browser Access

KasmVNC streams the desktop directly to any modern browser. No plugins or native clients.

🕑

Auto-Cleanup

Inactive VMs are automatically destroyed after configurable idle timeouts. Hard session caps enforced.

📈

Resource Caps

Each container is capped at 4 CPU cores and 8 GB RAM. Fair allocation across all tenants.

🔌

Dynamic Ports

Internal KasmVNC port 3000 is mapped to a free host port automatically. Zero config routing.

Workflow

How It Works

Three API calls. That's all it takes to give any user a full Linux desktop.

01

Create a VM

Call GET /api/create?developer_id=your_id to provision a new container. The response includes the URL to access the desktop.

02

Access the Desktop

Open the returned URL in any browser. KasmVNC streams the full Ubuntu desktop with keyboard, mouse, and clipboard support.

03

Automatic Cleanup

VMs self-destruct after 5 minutes of inactivity or 60 minutes total. Call GET /api/delete/{id} for manual removal.

API Endpoints

GET /api/health

Returns the current server status. Useful for uptime monitoring and load balancer health checks.

Response 
{"status": "ok"}
GET /api/create

Provisions a new Ubuntu VM container with KasmVNC. The container is automatically mapped to a free host port.

Parameters
developer_id string required Unique identifier for the site or developer requesting the VM.
site_limit integer Max concurrent VMs for this developer. Default: 5. Capped at MAX_VMS_PER_DEV.
delete_after integer Seconds of inactivity before auto-deletion. Capped at MAX_INACTIVITY_MINUTES.
premium string Premium code. If valid, the VM will never auto-delete.
Response 200 
{
  "status": "success",
  "container_id": "eb36b9c92cc...",
  "name": "vm-demo-a1b2c3d4",
  "port": 49156,
  "developer_id": "demo",
  "inactivity_timeout_seconds": 300,
  "max_session_minutes": 60,
  "premium": false,
  "message": "VM created successfully.",
  "url": "http://localhost:49156"
}
Example 
$ curl "http://localhost:8000/api/create?developer_id=myapp"
GET /api/list

Lists all running VMs. Requires admin authentication via query parameter or header.

Authentication
password string required Admin password via query (?password=) or X-Admin-Password header.
Parameters
developer_id string Filter results by developer ID.
Response 200 
{
  "status": "success",
  "vms": [
    {
      "id": "eb36b9c92cc...",
      "name": "vm-demo-a1b2c3d4",
      "status": "running",
      "port": 49156,
      "developer_id": "demo"
    }
  ]
}
DELETE /api/delete/{container_id}

Stops and removes a VM container. Requires admin authentication.

Path Parameters
container_id string required The Docker container ID to destroy.
Authentication
password string required Admin password via query or X-Admin-Password header.
Response 200 
{
  "status": "success",
  "message": "Container eb36b9c92cc stopped and removed"
}
